# 冷链运输联控日志命令行工具

供运营分析员在本地批量检查多辆冷藏车的油电冷机记录的命令行工具。

## 功能特点

- 📥 **导入行程文件** - 批量导入 CSV/JSON 格式的车机数据
- 🔍 **检查异常** - 按线路逐段检测异常情况
- 📋 **导出日报** - 按车队、司机、路线统计节油空间和温控风险
- ⏱️ **时间线展示** - 输入车牌查看详细时间线
- 📊 **多格式导出** - 支持 CSV 和 JSON 格式导出

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 准备数据

将车机导出的文件放入 `data/raw` 目录。支持以下格式：

**JSON 格式** (推荐):
```json
{
  "trip_id": "trip_001",
  "vehicle": {
    "plate": "京A12345",
    "fleet": "北京一队",
    "driver": "张师傅"
  },
  "date": "2025-06-20",
  "route": "北京-天津-北京",
  "segments": [...],
  "temperature_records": [...],
  "fuel_records": [...],
  "electric_records": [...],
  "cold_mode_records": [...],
  "ignition_records": [...],
  "charge_records": [...]
}
```

**CSV 格式**:
```csv
trip_id,车牌,车队,司机,日期,路线,记录类型,时间,...
```

### 3. 三个核心命令

#### 命令一：导入数据

```bash
# 导入 data/raw 目录下的所有文件
python main.py import

# 导入指定文件
python main.py import --file /path/to/file.json

# 导入前清空已有数据
python main.py import --clear
```

导入成功后会显示车辆和日期列表：

#### 命令二：检查异常

```bash
python main.py check
```

检查命令会：
1. 按线路逐段分析每辆车的行程
2. 检测以下异常：
   - 🔴 **装卸等待用油机** - 装货/卸货/等待期间使用油机制冷
   - 🔴 **插电未转电机** - 插电后未及时切换到电机制冷
   - 🔴 **模式切换温度波动** - 冷机模式切换前后温度波动过大
   - 🟡 **温度偏离设定** - 货厢温度偏离目标温度
   - 🟡 **插电时点火** - 插电状态下发动机点火

3. 显示异常统计信息
4. 支持输入车牌查看详细时间线

#### 命令三：导出日报

```bash
# 导出今天的日报
python main.py export

# 导出指定日期的日报
python main.py export --date 2025-06-20

# 只导出 CSV 格式
python main.py export --format csv

# 只导出 JSON 格式
python main.py export --format json
```

日报包含：
- 📊 **总体概览** - 异常总数、高风险数、节油潜力
- 🚛 **车队汇总** - 按车队统计油耗、电耗、风险
- 👷 **司机汇总** - 按司机统计（TOP 10 高风险）
- 🛣️ **路线汇总** - 按路线统计里程、平均油耗
- ⚠️ **重点异常** - TOP 10 高风险异常详情

导出的文件保存在 `data/exports` 目录：
- `daily_report_YYYY-MM-DD.csv` - 适合 Excel 打开
- `daily_report_YYYY-MM-DD.json` - 适合程序处理

## 检测的异常类型说明

| 异常类型 | 严重程度 | 说明 | 节油潜力 |
|---------|---------|------|---------|
| 装卸等待用油机 | 低/中 | 装货、卸货、等待期间使用油机而非电机 | ✅ |
| 插电未转电机 | 高 | 插电后10分钟内仍使用油机 | ✅ |
| 模式切换温度波动 | 中/高 | 冷机模式切换前后温度波动超过3℃ | ❌ |
| 温度偏离设定 | 中/高 | 货厢温度偏离目标温度超过3℃ | ❌ |
| 插电时点火 | 中 | 插电状态下发动机点火，存在安全隐患 | ✅ |

## 数据字段说明

### 必需字段
- `trip_id` - 行程唯一标识
- `plate` / `车牌` - 车牌号
- `fleet` / `车队` - 所属车队
- `driver` / `司机` - 司机姓名
- `date` / `日期` - 行程日期
- `route` / `路线` - 行驶路线

### 记录类型
- `temperature` / `温度` - 温度记录
- `fuel` / `油耗` - 油耗记录
- `electric` / `电耗` - 电耗记录
- `cold_mode` / `冷机模式` - 冷机模式切换
- `ignition` / `点火` - 发动机点火状态
- `charge` / `插电` - 电源插电状态
- `segment` / `路段` - 路段信息

## 配置参数

可在 `anomaly_detector.py` 中调整以下参数：
- `temp_fluctuation_threshold` - 温度波动阈值 (默认3℃)
- `mode_switch_window_minutes` - 模式切换检测窗口 (默认10分钟)
- `diesel_rate_per_hour` - 油机每小时油耗 (默认2.5L/h)

## 常见问题

**Q: 导入失败怎么办？**
A: 检查文件格式是否正确，确保必需字段存在。错误信息会显示具体的文件和原因。

**Q: 如何添加更多车辆数据？**
A: 将车机导出的 JSON 或 CSV 文件放入 `data/raw` 目录，然后执行 `import` 命令。

**Q: 可以调整异常检测的阈值吗？**
A: 可以，在 `cold_chain_monitor/anomaly_detector.py` 中修改 AnomalyDetector 类的初始化参数。

**Q: 日报可以自定义格式吗？**
A: 可以，修改 `cold_chain_monitor/report_generator.py` 中的 `_save_report_csv` 方法。
